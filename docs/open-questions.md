# 未解明事項ログ

形式上の未確定事項を、推測で公開APIへ固定しないための記録である。新しい証拠が得られた場合は、対象ファイル、作成アプリ版、比較条件を追記する。

## ブロックチェックサム

- 状態: 未解明
- 観測: データありブロックは非0、空ブロックは0。
- 不一致を確認済み: 圧縮前後のCRC-32、Adler-32、zlib末尾Adler-32。
- 現在の扱い: 検証せず不透明な `u32` として保持。
- 影響: 安全な書き込み対応を開始できない主要因。
- 次の調査: 1ピクセルだけ変更した最小ファイルを複数保存し、圧縮データと値の差分を比較する。

## `BlockStatus` の意味

- 状態: 未解明
- 観測: 値は0または1だが、データ有無とは一致しない。要素数は常にブロック数と一致。
- 現在の扱い: 不透明な `u32` として保持。
- 次の調査: 同じキャンバスで編集・保存・再保存を繰り返し、更新タイルとの相関を調べる。

## 12バイトのブロック属性

- 状態: 強い推定
- 観測値: `000500000000010000000100` のみ。
- 仮説: `u16 channels=5`, `u16 reserved=0`, `u32 width=256`, `u32 height=256`。
- 現在の扱い: 生バイトを保持しつつ、上記フィールドのgetterも提供。
- 次の調査: 1-bit、グレースケール、マスク、異なる形式のキャンバスを作成して比較する。

## DB参照はあるが外部チャンクがないID

- 状態: 未解明
- 観測: `Offscreen.BlockData` などの一部IDは `ExternalChunk` に存在しない。
- 仮説候補: 空・既定値、古い履歴、遅延生成、削除済みデータ。
- 現在の扱い: 高レベル解決時に「欠落」と「空」を区別する予定。
- 次の調査: 新規最小ファイル、レイヤー削除後、保存最適化前後で比較する。

## レイヤー種別と特殊機能

- 状態: 部分的
- 観測: `LayerType` は複数の数値を取り、サンプル間でスキーマ列にも差がある。
- 現在の扱い: フェーズ5では既知値を補助的に扱い、元の整数値を必ず保持する。
- 次の調査: ラスター、ベクター、テキスト、フォルダー、調整、3D、定規を1種類ずつ含む最小コーパスを作成する。
